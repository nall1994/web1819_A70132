RESUMO

Neste resumo, explicarei o que foi feito e onde se podem encontrar os resultados (os links estarão na secção de resultados).

No primeiro exercício utilizei um convertor online de XML para JSON e de seguida alterei o atributo "-id" para "_id", verificando no mongoDB se a inserção de documentos tinha a mesma contagem para cada uma das opções (todos os ids eram diferentes). O ficheiro devolvido pelo conversor veio imediatamente em array, o que facilitou a tarefa. Esse ficheiro encontra-se na pasta exercicio1 chamado de compositores.json

Para executar esta aplicação deverá colocar a funcionar o servidor de mongoDB, criando a base de dados testeEX1 e a colecção compositores e, de forma a preencher a base de dados deve executar o seguinte comando, na diretoria do exercicio 1:

mongoimport --db testeEX1 --collection compositores --type json --file compositores.json --jsonArray

Além disso, é necessário correr o servidor com npm start dentro da pasta do exercicio 1. Os pedidos deste exercício são escutados na porta 3000.

A partir deste momento, o servidor da API de dados está pronto a utilizar.

No segundo exercício, respondi primeiro às três perguntas pedidas. Essas respostas estão na pasta respostaQueries dentro da pasta exercicio2. query1.json refere-se à primeira pergunta e assim sucessivamente.

A resposta à primeira pergunta é 22. A resposta à segunda pergunta é 116 . A resposta à terceira pergunta é 58.

Finalmente, para executar a aplicação também desenvolvida no segundo exercício, basta executar o servidor dentro da pasta exercício 2 com npm start.

Este servidor do segundo exercício escuta pedidos na porta 3001.

Nota: A pasta node_modules foi apagada devido ao seu tamanho, em qualquer um dos casos terá que se fazer: npm install
